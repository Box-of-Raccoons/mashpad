# mashpad — design

**Date:** 2026-07-04
**Status:** Approved (brainstorm with Hardy)

## What it is

A lightweight BabySmash-style "baby keyboard smasher" for a Raspberry Pi 4 (1GB).
The Pi is a dedicated baby toy box: it boots straight into the app fullscreen,
the keyboard belongs to the baby, and a grown-up quits with a secret combo or SSH.

Replaces Scott Hanselman's BabySmash (WPF/.NET), which is too heavy for this
hardware and Windows-only.

## Platform & stack

- **Python 3 + pygame 2** on **Raspberry Pi OS Lite** (64-bit).
- Renders directly on the **KMS/DRM framebuffer** — no X, no Wayland, no
  desktop environment. Target footprint: well under 100MB RAM total.
- Runs as a **systemd service**: starts at boot, restarts on crash.
- **Dev mode flag** (`--windowed`) runs it in a normal window on Windows/desktop
  for development.

## Behavior

### Keyboard
- Letter key → that letter appears; digit key → that digit; any other key
  (space, enter, F-keys, modifiers alone, etc.) → a random shape (circle,
  square, star, triangle, heart, ...).
- Items spawn at a random position in a big bright color, scale in with a small
  bounce, linger, then fade out. The screen keeps at most ~20 live items.
- **Rate limiting:** key-repeat and mash-flooding are throttled so a held key
  or two-handed smash can't melt the Pi or flood the audio mixer.

### Audio
- On spawn: the item's spoken name ("A!", "Circle!", "Seven!") plus a random
  fun effect (pop/giggle/boing) from a bundled CC0 sound pack.
- **No runtime TTS.** The vocabulary is fixed (26 letters + 10 digits +
  ~8 shapes ≈ 44 words), so all name clips are pre-generated once at install
  time with **Piper** (natural neural voice) and played as WAV files at
  runtime. Nice voice, zero runtime TTS cost.

### Mouse / trackpad
- Mouse motion draws a fading rainbow trail.
- Clicks spawn a shape at the cursor.

### Grown-up controls
- **Ctrl+Alt+Q** quits the app (combo a baby won't mash). SSH is the fallback.
- No other UI: no menus, no settings screen. Configuration is constants in code.

## Structure

```
mashpad/
├── mashpad/
│   ├── main.py        # init, event loop, input handling, dev-mode flag
│   ├── sprites.py     # letter/digit/shape items, spawn/bounce/fade lifecycle
│   ├── audio.py       # clip loading + playback, effect randomization
│   └── gen_voice.py   # one-time Piper WAV generation for the 44 name clips
├── sounds/
│   ├── effects/       # CC0 pops/giggles/boings (bundled)
│   └── voice/         # generated by gen_voice.py at install (not committed)
├── install.sh         # Pi setup: apt/pip deps, Piper clip gen, systemd unit
├── mashpad.service    # systemd unit
└── tests/             # unit tests for the pure logic
```

## Testing

- **Unit tests** (pytest) for the pure logic: key→item mapping, spawn/fade
  lifecycle timing, live-item cap, rate limiting.
- **Rendering and audio** are verified by running the app: windowed on the dev
  machine first, then for real on the Pi (boot-to-app, smash test, exit combo).

## Name

`mashpad` — verified available 2026-07-04: no GitHub repos with that name,
PyPI free. Known unrelated squatters: an Urban Dictionary entry, a dormant
Twitter handle, a defunct Chilean shop ("MashPads"). Runner-up considered:
`keygiggle`. Rejected: `babykeys` (taken by multiple same-category projects).
